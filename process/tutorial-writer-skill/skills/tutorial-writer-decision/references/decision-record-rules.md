# Decision 阶段决策细则

## 本阶段职责

Decision 子技能不负责任何具体的决策项，而是管理**决策系统本身**：创建、查询、修改、冲突解决。

## 操作指引

### 查看决策状态

```
显示所有决策项的当前状态（pending/discussing/confirmed/deferred）
按分类分组，标注优先级
```

### 创建新决策

```json
{
  "id": "tech_new_item",
  "question": "问题描述",
  "type": "单选",
  "options": ["A", "B", "C"],
  "default": "A",
  "scope": "影响范围",
  "priority": "critical-high",
  "status": "pending",
  "created_at": "2026-05-29T00:00:00Z"
}
```

### 修改已有决策

1. 检查是否有下游依赖项受影响
2. 向用户展示影响评估
3. 用户确认后更新
4. 级联通知相关子技能

### 冲突解决

| 冲突类型 | 处理方法 |
|---------|---------|
| 新决策与已有矛盾 | 展示矛盾点，让用户选择覆盖或保留 |
| 用户指令与决策矛盾 | 提醒用户已有决策，提供修改选项 |
| 跨子技能决策不一致 | 统一归 decision 管理，各子技能只读 |
