# 跨章一致性规则

> 跨章节的技术栈、数据结构、代码质量、资源引用规则。这些规则由 review 子技能在 Q10-Q14 门禁中验证，但需要在 research 和 writing 阶段就遵守。

## R7: 跨章技术栈一致性

全书技术栈选型必须在 decision 子技能中统一确认，后续章节不得擅自偏离。

| 原则 | 说明 |
|------|------|
| 全书统一 | HTTP 库、编排框架、向量数据库等全书范围内保持一致 |
| 引入新工具 | 必须在章节开头说明"本章引入 XX 作为补充，完整技术栈选型见 decision-record" |
| 检测方式 | Grep 搜索全书框架引用，与决策记录交叉比对 |

## R8: 跨章数据结构兼容性

数据在章节间流动时（上一章输出 = 下一章输入），格式必须一致。

| 原则 | 说明 |
|------|------|
| 格式声明 | 涉及数据流转的章节开头必须声明输入/输出格式 |
| 格式变更 | 必须同步更新所有引用该数据的后续章节 |
| 检测方式 | 相邻章节数据格式对比 |

## R9: 代码质量标准

| 原则 | 说明 |
|------|------|
| 无逻辑错误 | 关键逻辑必须可运行或明确标注为伪代码 |
| 异步正确 | 异步上下文禁止使用同步 HTTP 调用 |
| 无死代码 | 无未使用的函数、变量、import |
| 示例数据 | 硬编码数据标注 `# 示例数据` |
| 检测方式 | 逐段审查代码逻辑 |

## R10: 网页友好资源引用

| 原则 | 说明 |
|------|------|
| 图片路径 | 使用 `./assets/images/xxx.png`，禁止穿透项目根目录 |
| 图片格式 | 优先 WebP，Fallback PNG，800-1200px 宽 |
| Mermaid | 节点 ≤15，语法经在线工具验证 |
| 跨章链接 | 使用相对路径，禁止绝对 URL |
| 代码行宽 | ≤80 字符 |
| 检测方式 | `mkdocs build` 无 error/warning |
