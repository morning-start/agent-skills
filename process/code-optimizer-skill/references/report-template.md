## 报告各部分详解

### 1. 总体评估

**目的**: 让读者快速了解代码整体质量

**评分标准**:

| 分数 | 质量等级 | 描述 |
|------|----------|------|
| 9-10 | 优秀 | 几乎无需改进 |
| 7-8 | 良好 | 少量改进点 |
| 5-6 | 中等 | 需要适度重构 |
| 3-4 | 较差 | 需要大量改进 |
| 1-2 | 糟糕 | 需要重写 |

**复杂度评级**:

| 等级 | 圈复杂度 | 描述 |
|------|----------|------|
| 低 | 1-10 | 简单清晰 |
| 中 | 11-15 | 适度复杂 |
| 高 | 16+ | 过于复杂 |

**示例**:

```markdown
### 1. 总体评估

- **代码质量评分**: 6/10
- **复杂度**: 高 (Cyclomatic Complexity: 18)
- **可维护性**: 中等
- **主要风险**: 
  - 🔴 SQL 注入漏洞（第 34 行）
  - 🟠 N+1 查询问题（第 23-28 行）
  - 🟡 重复代码（3 处相似逻辑）
- **技术债务**: 6-8 小时
```

### 2. 问题汇总

**目的**: 按优先级汇总所有问题，便于规划工作

**优先级定义**:

| 优先级 | 标识 | 描述 | 响应时间 |
|--------|------|------|----------|
| P0 | 🔴 | 严重 Bug/安全漏洞 | 立即 |
| P1 | 🟠 | 性能瓶颈/逻辑缺陷 | 24 小时 |
| P2 | 🟡 | 可维护性问题 | 下次迭代 |
| P3 | 🟢 | 优化建议 | 技术债务 |

**示例**:

```markdown
### 2. 问题汇总

| 优先级 | 问题类型 | 数量 | 影响范围 |
|--------|----------|------|----------|
| P0 | 严重 Bug | 1 | 核心支付逻辑 |
| P1 | 性能瓶颈 | 2 | 订单查询接口 |
| P2 | 可维护性 | 4 | 全局 |
| P3 | 优化建议 | 3 | 工具函数 |
```

### 3. 关键问题与重构方案

**目的**: 为每个问题提供详细的分析和可执行的重构方案

**必备要素**:

1. **位置**: 精确到行号
2. **描述**: 清晰说明问题
3. **影响**: 量化不优化的后果
4. **建议**: 具体的优化方案
5. **重构代码**: 含对比（优化前 vs 优化后）
6. **验证方法**: 如何验证重构正确性

**示例**:

```markdown
#### 🔴 问题 1: SQL 注入漏洞

- **位置**: 第 34 行
- **描述**: 直接使用字符串拼接 SQL 查询
  ```python
  # 问题代码
  query = f"SELECT * FROM users WHERE username = '{username}'"
  ```
- **影响**: 
  - 🔴 严重安全风险
  - 可能导致数据泄露、数据篡改
  - 攻击者可绕过认证
- **建议**: 立即使用参数化查询
- **重构代码**:
  ```python
  # ❌ 优化前
  def get_user(username: str) -> Optional[User]:
      query = f"SELECT * FROM users WHERE username = '{username}'"
      return db.execute(query)
  
  # ✅ 优化后
  def get_user(username: str) -> Optional[User]:
      query = "SELECT * FROM users WHERE username = ?"
      params = [username]
      return db.execute(query, params)
  ```
- **验证方法**: 
  1. 运行安全测试：`pytest tests/test_security.py`
  2. 尝试 SQL 注入攻击：`username = "admin' OR '1'='1"`
  3. 验证无法注入

```

### 4. 性能优化建议

**目的**: 提供专门的性能优化建议

**内容要求**:

- 当前复杂度（量化）
- 建议算法（对比）
- 预期提升（百分比）
- 代码示例（可执行）

**示例**:

```markdown
### 4. 性能优化建议

- **当前算法复杂度**: O(n²) - 双重循环查找重复项
- **建议算法**: O(n) - 使用哈希表
- **预期提升**: 99%+ (n=10000 时，从 100 秒降至 0.1 秒)
- **优化方案**:
  ```python
  # ❌ 优化前 - O(n²)
  def find_duplicates(items):
      duplicates = []
      for i in range(len(items)):
          for j in range(i + 1, len(items)):
              if items[i] == items[j]:
                  duplicates.append(items[i])
      return duplicates
  
  # ✅ 优化后 - O(n)
  def find_duplicates(items):
      seen = set()
      duplicates = set()
      for item in items:
          if item in seen:
              duplicates.add(item)
          else:
              seen.add(item)
      return list(duplicates)
  ```
```

## 报告使用场景

### 场景 1: 代码审查 (Code Review)

**重点**: 可维护性、代码规范

**模板调整**:
- 强调 L2 层问题（重复、命名、复杂度）
- 提供具体的重构代码
- 语气友好，避免指责

### 场景 2: 性能优化

**重点**: L3 层性能问题

**模板调整**:
- 详细分析算法复杂度
- 提供性能对比数据
- 包含基准测试方案

### 场景 3: 安全审计

**重点**: L3 层安全问题

**模板调整**:
- 完整的安全检查清单
- 每个漏洞的详细分析
- 包含安全测试用例

### 场景 4: 技术债务清理

**重点**: 量化债务、优先级排序

**模板调整**:
- 详细的技术债务评估
- 债务构成分析
- 清理计划和时间表

## 报告质量检查清单

生成报告前，检查以下项目：

- [ ] 总体评估完整（评分、复杂度、风险）
- [ ] 问题汇总表格清晰
- [ ] 每个问题都有位置（行号）
- [ ] 每个问题都有重构代码示例
- [ ] 优先级标注正确
- [ ] 影响量化（性能提升百分比）
- [ ] 安全检查清单完整
- [ ] 后续行动计划可执行
- [ ] 语气专业且友好
- [ ] 代码示例可复制执行

## 🛡️ 代码优化分析报告

### 1. 总体评估

- **复杂度**: 高 (Cyclomatic Complexity: 16)
- **可维护性**: 低
- **主要风险**: 
  - 🔴 SQL 注入漏洞（第 34 行）
  - 🟠 N+1 查询问题（第 23-28 行）
  - 🟡 严重代码重复（3 处）
- **技术债务**: 8-10 小时

### 2. 问题汇总

| 优先级 | 问题类型 | 数量 | 影响范围 |
|--------|----------|------|----------|
| P0 | 严重 Bug | 1 | 用户认证模块 |
| P1 | 性能瓶颈 | 2 | 订单查询接口 |
| P2 | 可维护性 | 5 | 全局 |
| P3 | 优化建议 | 2 | 工具函数 |

### 3. 关键问题与重构方案

#### 🔴 问题 1: SQL 注入漏洞

- **位置**: 第 34 行
- **描述**: 直接使用字符串拼接 SQL 查询
- **影响**: 严重安全风险，可能导致数据泄露
- **建议**: 立即使用参数化查询
- **重构代码**:
  ```python
  # ❌ 优化前
  query = f"SELECT * FROM users WHERE username = '{username}'"
  
  # ✅ 优化后
  query = "SELECT * FROM users WHERE username = ?"
  params = [username]
  db.execute(query, params)
  ```

#### 🟠 问题 2: N+1 查询问题

- **位置**: 第 23-28 行
- **描述**: 循环内执行数据库查询
- **影响**: 查询次数爆炸（2n 次），n=100 时执行 200 次查询
- **建议**: 改为批量查询
- **重构代码**:
  ```python
  # ❌ 优化前
  for user_id in user_ids:
      user = db.query("SELECT * FROM users WHERE id = ?", user_id)
  
  # ✅ 优化后
  users = db.query(
      "SELECT * FROM users WHERE id IN (?)",
      user_ids
  )
  ```

### 4. 性能优化建议

- **当前算法复杂度**: O(n²)
- **建议算法**: O(n log n)
- **预期提升**: 95%+ (n=10000 时)

### 5. 安全性检查清单

- [x] SQL 注入（已修复）
- [ ] XSS 攻击（待检查）
- [ ] CSRF 保护（待检查）
- [ ] 硬编码密钥（待检查）
