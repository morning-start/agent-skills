# 章节长度自动规划规则

> 本文档定义了教程撰写过程中章节长度的自动判断和规划方法，三级判断体系：阶段分级 → 小节数微调 → 密度因子修正。

---

## 1. 按阶段自动分级

### 1.1 阶段判定依据

| 阶段 | 章节 | 内容特征 | 默认级别 | 目标行数 | 目标字数 |
|------|------|---------|---------|---------|---------|
| **认知篇** | 建立认知的章节 | 概念为主，需建立理解但不展开实现细节 | 标准 | 200-300 | 8K-15K |
| **基础篇** | 核心实操章节 | 含大量代码示例、配置步骤、实操指导 | 详细 | 350-500 | 15K-25K |
| **进阶篇** | 优化策略章节 | 偏原理讲解、配置对比、策略选择 | 标准 | 250-350 | 10K-15K |
| **生产篇** | 企业级部署章节 | 架构设计、Docker/K8s、监控运维 | 标准 | 250-400 | 10K-18K |
| **前沿篇** | 趋势展望章节 | 最新动态、行业案例、未来方向 | 精简 | 180-280 | 7K-12K |

### 1.2 快速判定决策树

```
该章节的核心任务是什么？
│
├─ 让读者"理解"某个概念/现象
│  └─ → 认知篇 → 标准 → 200-300 行
│
├─ 教读者"动手做"某件事（含代码/步骤）
│  └─ → 基础篇 → 详细 → 350-500 行
│
├─ 讲"怎么做得更好"（优化/调优/进阶）
│  └─ → 进阶篇 → 标准 → 250-350 行
│
├─ 讲"怎么部署到生产环境"
│  └─ → 生产篇 → 标准 → 250-400 行
│
└─ 讲"未来趋势/最新进展"
   └─ → 前沿篇 → 精简 → 180-280 行
```

## 2. 按小节数量微调

### 2.1 微调公式

```
调整后目标 = 阶段基准中值 × (1 + 小节系数)

其中：
  阶段基准中值 = (目标行数下限 + 目标行数上限) / 2
  小节系数见下表
```

### 2.2 小节系数表

| 小节数量 | 系数 | 调整幅度 | 说明 |
|---------|------|---------|------|
| ≤3 个 | 0.0 | ±0% | 内容紧凑，不需额外空间 |
| 4-6 个 | +0.2 | +20% | 需要适度展开每个小节 |
| 7-9 个 | +0.4 | +40% | 小节较多，每个都需要充分论述 |
| ≥10 个 | +0.6 | +60% | 内容丰富，需要充足篇幅 |

### 2.3 计算示例

**示例 A**：Ch4 基础篇，5 个小节（4.1-4.4）
- 阶段基准：(350+500)/2 = 425 行
- 小节系数：5 个小节 → +20%
- 目标：425 × 1.2 = **510 行**

**示例 B**：Ch12 前沿篇，3 个小节（12.1-12.3）
- 阶段基准：(180+280)/2 = 230 行
- 小节系数：3 个小节 → +0%
- 目标：230 × 1.0 = **230 行**

## 3. 内容密度因子修正

### 3.1 密度因子表

在阶段分级和小节数微调的基础上，根据内容特征进一步调整：

| 因素 | 调整幅度 | 触发条件 | 说明 |
|------|---------|---------|------|
| 含完整代码示例 | +20% | 有 ≥ 3 个可运行的代码块 | 代码占用大量行数 |
| 含多张对比表格 | +15% | 表格数量 ≥ 4 张 | 表格增加信息密度但也增加行数 |
| 含大量数据支撑 | +10% | 引用数据点 ≥ 8 处 | 数据论证需要额外空间 |
| 含架构图/流程描述 | +10% | 需要文字描述架构流程 | 架构描述占用篇幅 |

### 3.2 叠加计算

多个密度因子可叠加（理论上限 +40%，防止过度膨胀）：

```
最终目标 = 阶段基准 × (1 + 小节系数) × (1 + Σ密度因子)

示例：基础篇章节，5个小节，含 5 个代码块，4 张表格
= 425 × 1.2 × (1 + 0.20 + 0.15)
= 425 × 1.2 × 1.35
= 688 行（上限）
```

### 3.3 密度因子使用注意事项

- **不重复计算**：同一内容特征只计一次因子
- **主观判断**：因子触发与否由 Agent 自行判断，WRITE 阶段实时感知
- **上限约束**：总调整幅度不超过基础值的 +60%

## 4. 用户覆盖指令

用户可在触发撰写时指定长度偏好，覆盖自动计算的默认值：

| 指令 | 乘数 | 效果 | 适用场景 |
|------|------|------|---------|
| "精简版" / "简写" / "快速版" | × 0.6 | 大幅压缩 | 时间紧、快速过一遍 |
| "标准版"（默认） | × 1.0 | 正常输出 | 正常写作流程 |
| "详细版" / "深入展开" / "全面版" | × 1.4 | 充分展开 | 核心重点章节、读者零基础 |

**应用方式**：

```
最终目标 = 自动计算值 × 用户乘数

示例：自动计算 500 行，用户说"写个精简版"
→ 最终目标 = 500 × 0.6 = 300 行
```

## 5. WRITE 阶段实时长度感知

### 5.1 检查点机制

撰写过程中，每完成一个小节后检查累计行数：

| 累计进度 | 判断 | 行动 |
|---------|------|------|
| ≤ 60% 目标 | 正常初期 | 继续正常节奏 |
| 60%-80% 目标 | 进入中期 | 保持节奏，关注剩余小节的篇幅分配 |
| 80%-100% 目标 | 进入收尾 | 开始收尾，聚焦核心要点，控制新内容的展开深度 |
| 100%-120% 目标 | 轻微超标 | 评估后续小节是否可以精简 |
| > 120% 目标 | **过长预警** | 必须采取措施（见下方） |

### 5.2 过长预警应对策略

当累计行数超过目标 120% 时，按以下优先级处理：

| 优先级 | 策略 | 适用场景 |
|-------|------|---------|
| 1 | 精简冗余段落 | 存在过渡性文字或重复表述 |
| 2 | 合并相似小节 | 两个小节内容有重叠 |
| 3 | 将独立大节拆为子文件 | 单节 > 100 行且有独立完整性 |
| 4 | 降低密度因子预期 | 减少代码示例或表格数量 |
| 5 | 与用户确认 | 以上策略均不可行时 |

### 5.3 过短预警应对策略

当预计完成时行数低于目标 80% 时：

| 优先级 | 策略 | 适用场景 |
|-------|------|---------|
| 1 | 检查遗漏要点 | 对照规划确认是否遗漏小节 |
| 2 | 增加代码示例 | 适当补充可运行的完整示例 |
| 3 | 增加对比表格 | 将文字描述转为表格对比 |
| 4 | 深化现有内容 | 对关键概念增加更多层次的分析 |
| 5 | 与用户确认 | 内容确实不需要更多篇幅时 |
