## 概览

需求优先级排序是个人项目成功的关键，过度规划会导致项目无法交付，而规划不足会导致交付后无人使用。

## 优先级定义

### P0 - 必须有（Must Have）

**定义**：没有这个功能，项目就没有存在价值

**特征**：
- 核心用户场景的最低保障
- 竞品都有，缺了会被用户放弃
- 技术/商业风险最高

**示例**：
```
效率工具 MVP：
├── 全局快捷键唤起
├── 搜索输入
└── 至少 1 个可用插件
```

### P1 - 应该有（Should Have）

**定义**：对用户体验有显著提升，但没有也不会导致用户流失

**特征**：
- 差异化竞争的要点
- 用户期望的"加分项"
- 开发成本中等

**示例**：
```
├── 剪贴板历史
├── 模糊搜索
└── 快捷键自定义
```

### P2 - 可以有（Could Have）

**定义**：有则更好，无也可接受

**特征**：
- 增强用户满意度的功能
- 可以后续迭代补充
- 开发成本较低

**示例**：
```
├── 主题定制
├── 插件市场
└── 云端同步
```

### P3 - 不会有（Won't Have）

**定义**：当前版本明确不做

**原因**：
- 技术难度过高
- 与核心定位不符
- 可以长期规划

**示例**：
```
├── AI Agent 集成（MVP 后）
├── 插件签名验证（MVP 后）
└── macOS 版本（MVP 后）
```

## 决策框架

### MoSCoW 方法

| 优先级 | 含义 | 占比建议 |
|--------|------|----------|
| Must have | 必须有 | 60% |
| Should have | 应该有 | 20% |
| Could have | 可以有 | 15% |
| Won't have | 不会有 | 5% |

### 个人项目调整

由于个人开发资源有限，建议调整为：

| 优先级 | 含义 | 占比建议 |
|--------|------|----------|
| P0 | MVP 核心 | 50% |
| P1 | MVP 完成后的第一梯队 | 30% |
| P2 | 后续迭代 | 20% |

## MVP 边界定义

### MVP 核心原则

1. **最小功能集**：只做满足核心场景的最少功能
2. **快速验证**：能快速发布给用户验证假设
3. **迭代基础**：为后续迭代奠定基础

### MVP 边界检查清单

```
□ 这个功能解决什么用户问题？
□ 没有这个功能，用户会怎么做？
□ 这个功能的技术复杂度如何？
□ 这个功能需要多少开发时间？
□ 这个功能是否可以后续迭代补充？
```

### MVP 示例：效率工具

**P0（5 个功能）**：
```
1. 全局快捷键唤起
2. 搜索输入
3. 快捷命令插件
4. 剪贴板增强插件
5. 文件搜索插件
```

**P1（4 个功能）**：
```
6. 计算器插件
7. 二维码插件
8. 截图增强插件
9. 快捷键自定义
```

**P2（2 个功能）**：
```
10. 云端同步配置
11. 主题定制
```

## 优先级排序方法

### 1. 价值-成本矩阵

| | 低成本 | 高成本 |
|---|---|---|
| **高价值** | 优先做 | 快速做 |
| **低价值** | 快速做 | 最后做 |

### 2. 依赖关系排序

```
功能 A（前置）
    │
    ├──→ 功能 B（依赖 A）
    │
    └──→ 功能 C（独立）
```

### 3. 技术风险排序

| 风险等级 | 策略 | 示例 |
|----------|------|------|
| 高风险 | MVP 后验证 | WASM 补丁 |
| 中风险 | P1 阶段做 | AI 集成 |
| 低风险 | P0 先做 | 基础框架 |

## 使用示例

### 示例：确定 MVP 边界

**背景**：开发一个效率工具

**讨论问题 1**：QuickJS 插件还是 Webview 插件先做？

| 因素 | QuickJS | Webview |
|------|---------|---------|
| 开发成本 | 低 | 高 |
| 性能 | 快 | 慢 |
| 功能上限 | 低 | 高 |

**决策**：MVP 先做 QuickJS（低风险快速验证）

**讨论问题 2**：WASM 补丁是否包含？

| 因素 | 包含 | 不包含 |
|------|------|--------|
| MVP 复杂度 | 高 | 低 |
| AI 能力 | 有 | 无 |
| 后续迁移 | 简单 | 无需 |

**决策**：MVP 包含 WASM（crypto + AI 各一个）

**最终 MVP 边界**：
```
P0：QuickJS 基础框架 + 5 个 QuickJS 插件 + 1 个 Webview 插件
```
