# 关键检测维度清单 (Checklist)

## 概览

本清单包含代码优化专家技能的全部检测维度，按照三层分析漏斗架构组织。

---

## L1: 静态合规层检测清单

### 命名规范

| 检查项 | 描述 | 示例 | 修复方案 |
|--------|------|------|----------|
| **变量命名** | 变量名是否有语义 | `let d = 10` → `let daysUntilExpiry = 10` | 使用描述性名称 |
| **函数命名** | 函数名是否表达意图 | `function calc(p, r)` → `function calculatePriceWithTax(price, taxRate)` | 动词 + 名词结构 |
| **类命名** | 类名是否使用大驼峰 | `class userController` → `class UserController` | PascalCase |
| **常量命名** | 常量是否全大写 | `const max_count = 100` → `const MAX_COUNT = 100` | UPPER_SNAKE_CASE |
| **布尔变量** | 布尔值是否包含 is/has/can 等前缀 | `let ready = true` → `let isReady = true` | 添加系动词前缀 |

### 注释完整性

| 检查项 | 描述 | 标准 | 工具 |
|--------|------|------|------|
| **公共 API** | 公开函数/类是否有文档注释 | JSDoc/Docstring 完整 | ESLint, Pylint |
| **复杂逻辑** | 复杂算法是否有说明注释 | 解释"为什么"而非"做什么" | 人工审查 |
| **参数说明** | 函数参数是否有类型和用途说明 | Args/Returns 完整 | TypeDoc, Sphinx |
| **TODO 标记** | 临时代码是否有 TODO 注释 | 包含责任人和截止日期 | 代码审查 |

### 未使用代码检测

| 类型 | 检测方法 | 风险 | 处理建议 |
|------|----------|------|----------|
| **未使用变量** | 变量声明后未引用 | 内存浪费 | 删除或添加使用 |
| **未使用函数** | 函数定义后未调用 | 代码冗余 | 删除或导出 |
| **未使用导入** | import 但未使用 | 打包体积增加 | 删除导入 |
| **未使用参数** | 函数参数未使用 | API 误导 | 删除或标记 `_` |

### 死代码识别

| 类型 | 示例 | 检测方法 |
|------|------|----------|
| **无法到达的代码** | `return` 后的语句 | 控制流分析 |
| **永远为假的条件** | `if (false) { ... }` | 常量折叠 |
| **空的条件分支** | `if (condition) {}` | AST 分析 |
| **未导出的私有代码** | 未使用的私有函数 | 引用计数 |

### 语法风险

| 风险类型 | 示例 | 修复方案 |
|----------|------|----------|
| **隐式类型转换** | `if (x == "5")` | 使用 `===` 严格相等 |
| **浮点数比较** | `if (a + b === 0.3)` | 使用误差范围比较 |
| **原型污染** | `obj[__proto__] = ...` | 使用 Object.create(null) |
| **竞态条件** | 异步操作未等待 | 添加 await/async |

## L2: 逻辑与结构层检测清单

### 代码重复 (DRY 原则)

| 重复类型 | 识别标准 | 优化手段 | 优先级 |
|----------|----------|----------|--------|
| **字面重复** | 完全相同的代码块 | 提取为函数 | P1 |
| **结构重复** | 相似的模式，仅变量名不同 | 参数化/模板方法 | P1 |
| **逻辑重复** | 相同的业务逻辑在不同位置 | 提取为公共模块 | P1 |
| **测试重复** | 相似的测试用例 | 参数化测试/辅助函数 | P2 |

**检测阈值**: 相似度 > 80%，重复行数 ≥ 5 行

### 函数/类长度

| 指标 | 阈值 | 影响 | 优化方案 |
|------|------|------|----------|
| **函数行数** | > 50 行 | 难以理解、测试困难 | 拆分为多个小函数 |
| **类行数** | > 300 行 | 职责不清、耦合度高 | 提取子类/使用组合 |
| **文件行数** | > 500 行 | 导航困难 | 拆分为多个模块 |
| **方法数量** | > 20 个/类 | 接口复杂 | 拆分接口/使用策略模式 |

### 魔法数字/字符串

| 类型 | 示例 | 问题 | 修复方案 |
|------|------|------|----------|
| **魔法数字** | `if (status == 500)` | 缺乏语义 | `if (status === HttpStatus.INTERNAL_SERVER_ERROR)` |
| **魔法字符串** | `if (type === "vip")` | 易拼写错误 | `if (type === UserType.VIP)` |
| **魔法数组** | `if ([1, 3, 5].includes(status))` | 意图不明 | 定义为常量 `VALID_STATUSES` |
| **魔法对象** | `const config = { timeout: 5000 }` | 分散定义 | 集中配置管理 |

### 循环嵌套深度

| 嵌套层数 | 可读性 | 建议 |
|----------|--------|------|
| **1-2 层** | ✅ 良好 | 保持 |
| **3 层** | 🟡 可接受 | 考虑优化 |
| **4 层+** | 🔴 糟糕 | 必须重构 |

**优化手段**:

- 提前返回 (Early Return)
- 卫语句 (Guard Clauses)
- 提取方法 (Extract Method)
- 使用流式 API (map/filter/reduce)
- 使用多态替代条件判断

### 圈复杂度 (Cyclomatic Complexity)

| 复杂度范围 | 评级 | 建议 |
|------------|------|------|
| **1-10** | ✅ 低 | 保持 |
| **11-15** | 🟡 中 | 注意 |
| **16-20** | 🟠 高 | 需要重构 |
| **21+** | 🔴 极高 | 立即重构 |

**计算方法**: `CC = 决策点数量 + 1`

**决策点包括**: `if`, `else if`, `else`, `for`, `while`, `case`, `catch`, `&&`, `||`, `?:`

### 抽象不足

| 症状 | 示例 | 重构方案 |
|------|------|----------|
| **大量 if-else** | 10+ 个条件分支 | 策略模式/状态模式 |
| **类型判断** | `if (type === 'A') ... else if (type === 'B')` | 多态/工厂模式 |
| **重复的 switch** | 多处相同的 switch 语句 | 使用映射表/策略模式 |
| **过程式代码** | 长流程的线性代码 | 面向对象/函数组合 |

## L3: 性能与安全层检测清单

### 算法复杂度

| 场景 | 低效方案 | 高效方案 | 提升幅度 |
|------|----------|----------|----------|
| **查找** | 线性查找 O(n) | 哈希表 O(1) | 显著 |
| **排序** | 冒泡排序 O(n²) | 快速排序 O(n log n) | 巨大 |
| **去重** | 双重循环 O(n²) | Set/Map O(n) | 巨大 |
| **连接字符串** | 循环拼接 O(n²) | join/array O(n) | 显著 |
| **数组查找** | includes O(n) | Set.has O(1) | 显著 |

### 数据库性能

| 问题 | 示例 | 影响 | 优化方案 |
|------|------|------|----------|
| **N+1 查询** | 循环内查询 | O(n) 次查询 | 批量查询/JOIN |
| **缺少索引** | WHERE 无索引字段 | 全表扫描 | 添加索引 |
| **SELECT *** | 查询所有列 | 网络开销大 | 只查需要的列 |
| **大事务** | 事务包含过多操作 | 锁竞争激烈 | 拆分事务 |
| **隐式转换** | 字符串字段传数字 | 索引失效 | 类型匹配 |

### 内存管理

| 问题 | 风险 | 检测点 | 修复方案 |
|------|------|--------|----------|
| **内存泄漏** | 内存持续增长 | 未清理的定时器/监听器 | 及时清理 |
| **大对象未释放** | GC 压力大 | 缓存无上限 | LRU 缓存/弱引用 |
| **闭包陷阱** | 意外引用大对象 | 循环中的闭包 | 避免不必要的闭包 |
| **数组预分配** | 动态扩容开销 | 大数组 push | 预分配容量 |

### I/O 性能

| 问题 | 影响 | 优化方案 |
|------|------|----------|
| **同步 I/O** | 阻塞主线程 | 异步 I/O |
| **小文件频繁读写** | 系统调用开销大 | 缓冲流/批量操作 |
| **未使用流式处理** | 大文件内存占用高 | 使用 Stream |
| **重复读取** | 磁盘 I/O 浪费 | 缓存读取结果 |

### 网络安全

| 漏洞 | 风险等级 | 检测方法 | 修复方案 |
|------|----------|----------|----------|
| **SQL 注入** | 🔴 严重 | 字符串拼接 SQL | 参数化查询/ORM |
| **XSS 攻击** | 🔴 严重 | 直接输出用户输入 | HTML 转义/CSP |
| **CSRF** | 🟠 高 | 无 Token 验证 | CSRF Token/SameSite |
| **路径遍历** | 🟠 高 | 用户输入拼接路径 | 白名单验证/sanitize |
| **命令注入** | 🔴 严重 | shell 执行含用户输入 | 避免 shell/参数化 |

### 密钥管理

| 问题 | 风险 | 示例 | 修复方案 |
|------|------|------|----------|
| **硬编码密码** | 🔴 严重 | `const password = "admin123"` | 环境变量/密钥管理 |
| **硬编码 API Key** | 🔴 严重 | `const apiKey = "sk-xxx"` | 配置文件/.env |
| **明文存储密钥** | 🔴 严重 | 代码中明文密钥 | 加密存储/KMS |
| **密钥提交 Git** | 🔴 严重 | .gitignore 缺失 | 添加.gitignore/密钥扫描 |

### 并发安全

| 问题 | 风险 | 场景 | 修复方案 |
|------|------|------|----------|
| **竞态条件** | 高 | 多线程共享资源 | 加锁/原子操作 |
| **死锁** | 高 | 循环等待锁 | 锁顺序/超时机制 |
| **数据竞争** | 高 | 无保护的状态修改 | 不可变数据/锁 |
| **线程不安全集合** | 中 | 多线程访问集合 | 并发集合/同步包装 |

## 检测优先级规则

### P0: 严重问题（立即修复）

- 🔴 SQL 注入、命令注入
- 🔴 硬编码密钥/密码
- 🔴 严重的内存泄漏
- 🔴 死锁风险
- 🔴 数据竞争

### P1: 高优先级（24 小时内）

- 🟠 N+1 查询问题
- 🟠 O(n²) 算法（大数据集）
- 🟠 严重的代码重复
- 🟠 函数/类过长
- 🟠 缺少异常处理

### P2: 中优先级（下次迭代）

- 🟡 魔法数字/字符串
- 🟡 命名不规范
- 🟡 注释缺失
- 🟡 嵌套过深（3-4 层）
- 🟡 未使用变量/导入

### P3: 低优先级（技术债务）

- 🟢 代码风格问题
- 🟢 死代码
- 🟢 轻微的重复
- 🟢 优化建议（非必需）

## 使用指南

### 如何使用本清单

1. **L1 扫描**: 逐行检查静态合规问题
2. **L2 分析**: 识别结构和逻辑问题
3. **L3 检测**: 深度分析性能和安全
4. **优先级排序**: 按 P0→P1→P2→P3 排序
5. **生成报告**: 输出问题列表和重构方案
