# 写作中自检清单

> 每完成一个小节后执行，避免到 REVIEW 阶段才发现大量问题。

## 即时检查项

### 内容检查

```
□ 本小节是否覆盖了规划中的所有要点？
□ 是否有游离于规划之外的内容？
□ 先讲原理再给实践（Why → What → How）？
□ 复杂概念是否有比喻或举例说明？
```

### 数据与引用

```
□ 文中所有数字有来源标注？
□ 引用格式是否统一？（来源: [描述](路径)）
□ 对比结论是否有数据支撑？
```

### 格式检查

```
□ 代码块有语言标注？
□ 标题层级正确（# → ## → ###）？
□ 表格头尾对齐？
□ 当前小节行数是否合理（< 100 行）？
```

## 每节完成后的累计检查

```bash
# 累计行数 vs 目标行数
#   60% → 正常节奏
#   80% → 开始收尾
#  120% → 过长预警，考虑拆分
```

## 进入 REVIEW 前的最终检查

```
□ 所有小节已完成
□ 无 TODO / 待补充 / 占位符
□ 代码无逻辑错误
□ 引用来源全部有效
□ 资料已归档到 reference/
```
