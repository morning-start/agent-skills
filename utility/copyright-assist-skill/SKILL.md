# 软件著作权申请辅助技能

## 概述

本技能辅助用户在中国版权保护中心（https://www.ccopyright.com.cn/）完成计算机软件著作权登记申请（R11表单）。覆盖从账号注册到下证的全流程，提供源代码提取、说明书撰写、材料打包等自动化工具。

**官方平台**：中国版权保护中心 https://www.ccopyright.com.cn/

**核心要求**：
- 源代码文档：前30页 + 后30页，每页不少于50行，总计不少于3000行
- 说明书文档：操作手册/用户说明书/设计说明书（三选一），一般15页以上，建议60页
- 仅支持PDF格式上传
- 签章页需手写签名后扫描PDF上传（不可用电子签名）
- 受理后1-2周，通常2-3个月内下证；建议提前3个月申请

## 当何时使用

- 用户需要申请软件著作权
- 用户已有项目代码，需要提取和格式化源代码
- 用户需要撰写操作手册、用户说明书或设计说明书
- 用户需要检查软著申请材料的完整性
- 用户需要将材料打包为符合规范的ZIP

## 前置准备

### 个人申请所需材料
1. 常用手机号（仅可绑定一次）
2. 常用邮箱（仅可绑定一次）
3. 身份证正反面清晰照片
4. 手持身份证正面半身彩色照片

### 企业申请所需材料
1. 营业执照副本原件彩色扫描件或照片
2. 经办人身份证正反面清晰照片
3. 加盖公章的《授权书》扫描件（模板在实名认证页面可下载）
4. 经办人手机号
5. 经办人邮箱

## 操作流程

### 步骤1：账号注册与实名认证

**平台入口**：https://www.ccopyright.com.cn/

1. 点击注册，选择类型：个人申请选【个人】，企业/学校/代理选【机构】
2. 填写用户名、设置密码、手机号验证
3. 点击【去登记】→ 左侧【计算机软件著作权相关登记】
4. 完善账号信息，绑定邮箱
5. 填写个人实名信息并确认
6. 等待审核（1-3个工作日）

**注意**：首次使用有引导提示，可跳过。

### 步骤2：填写申请表（R11表单）

登录后进入版权登记 → 软件登记 → R11【计算机软件著作权登记申请】。

#### 2.1 基本信息

| 字段 | 说明 | 默认值 |
|------|------|--------|
| 取得方式 | 原始取得/继受取得 | 原始取得 |
| 权利范围 | 全部权利/部分权利 | 全部权利 |
| 软件全称 | 仔细填写，错误会导致补正 | — |
| 软件简称 | 可选 | — |
| 版本号 | 如 V1.0 | — |

**软件全称命名规范**：
- 建议根据功能以 **"软件"**、**"系统"** 或 **"平台"** 结尾
- 示例：`智能用户管理软件`、`数据分析系统`、`在线协作平台`
- **一致性要求**：软件全称必须在以下所有位置保持一致：
  - 申请表中的软件全称字段
  - 开发目的描述
  - 面向领域描述
  - 主要功能描述
  - 技术特点描述
  - 说明书/操作手册页眉
  - 源代码文档页眉
- 填写前务必确认名称，错误会导致补正，影响下证时间

**继受取得**适用于：通过受让、继承或承受方式取得著作权的情况。

#### 2.2 软件开发信息

| 字段 | 选项 | 个人独立申请推荐 |
|------|------|-----------------|
| 软件分类 | 应用软件/操作系统/嵌入式软件/中间件 | 应用软件 |
| 软件说明 | 原创/修改（含翻译、合成） | 原创 |
| 开发方式 | 独立开发/合作开发/委托开发/下达任务开发 | 独立开发 |
| 完成日期 | 应在提交日期之前几日 | — |
| 发表状态 | 未发表/已发表 | 未发表 |

**软件分类说明**：
- **应用软件**：微信、Photoshop、财务系统、APP等用户直接使用的软件
- **操作系统**：Windows、Linux、Android、iOS
- **嵌入式软件**：运行在芯片、传感器、控制器内部
- **中间件**：数据库中间件、消息队列（Kafka）、Web服务器（Tomcat）

**开发方式说明**（除独立开发外均需提交合同/协议）：
- 合作开发：需提交合作开发合同，签订日期应在开发完成日期前3个月以上（建议6个月前）
- 委托开发：需提交委托开发合同
- 下达任务开发：需提交项目任务书或合同

#### 2.3 软件功能与特点

根据项目实际开发情况撰写，需贴合操作手册和代码内容。此部分内容较多，需认真填写。

### 步骤3：准备鉴别材料

软著申请需上传2份PDF材料：**源代码文档**和**说明书文档**。

#### 3.1 源代码提取

调用脚本自动提取和格式化源代码：

```bash
python scripts/extract_source_code.py --code-dir ./src --output ./formatted_code.txt
```

**脚本功能**：
- 自动判断总行数，决定提取策略：
  - 代码总行数 < 3000行：全量提交所有代码
  - 代码总行数 ≥ 3000行：提取前30页 + 后30页
- 每页强制补足50行（含空行和注释）
- 自动用注释填充空白页
- 添加页码标记
- 支持多版本：`--version 1.0.0`

**代码格式要求**：
- 每页不少于50行
- 程序每页不少于50行，文档每页不少于30行
- 代码总行数不足60页的，提交全部源程序
- 最后1页结尾一定要是完成的功能模块

详细格式规范见 [references/source-code-format.md](references/source-code-format.md)。

#### 3.2 说明书撰写

三选一：设计说明书 / 用户说明书 / 操作手册。

**选择建议**：
- 软件面向普通用户、操作简单 → **用户说明书**（首选）
- 软件功能复杂、多模块配置 → **操作手册**
- 侧重展示设计和实现 → **设计说明书**

**撰写指导**：
- 设计说明书：[references/design-manual-guide.md](references/design-manual-guide.md)
- 用户说明书：[references/user-manual-guide.md](references/user-manual-guide.md)
- 操作手册：[references/operation-manual-guide.md](references/operation-manual-guide.md)

**模板**：
- 设计说明书：[assets/templates/design-manual-template.md](assets/templates/design-manual-template.md)
- 用户说明书：[assets/templates/user-manual-template.md](assets/templates/user-manual-template.md)

**页数要求**：一般以15页以上为佳，建议60页以上。

### 步骤4：资源完整性检查

```bash
python scripts/check_resources.py --code-dir ./src --doc-dir ./docs --screenshot-dir ./screenshots --output report.json
```

**检查项目**：
- 代码行数、文件数量是否充足
- 截图数量和分辨率是否达标
- 文档是否完整
- 生成JSON格式检查报告

### 步骤5：打印签章页

1. 在版权中心下载签章页
2. 手写签名（不可用电子签名或电子章）
3. 扫描为PDF文件

### 步骤6：打包提交材料

```bash
python scripts/package_submission.py \
  --software-name "用户管理系统" \
  --version 1.0.0 \
  --manual ./用户说明书.pdf \
  --code ./formatted_code.txt \
  --output ./output
```

**打包功能**：
- 自动生成符合命名规范的ZIP包（如 `用户管理系统_1.0.0.zip`）
- 重命名文件为标准格式
- 生成资源清单文件
- 验证包结构完整性

### 步骤7：提交与查询

1. 在用户中心上传ZIP包
2. 查询申请状态：受理 → 审查 → 制发证
3. 自受理日起60日内审查完成
4. 实际周期：提交后1-2周受理，2-3个月内下证

## 多版本申请

同一软件多个版本（V1.0, V2.0）的独立申请：

1. 按版本组织资源目录（`src/`, `src_v2/`, `screenshots/v1.0/`, `screenshots/v2.0/`）
2. 每个版本独立提取代码、生成说明书
3. 分别打包：`用户管理系统_1.0.0.zip`、`用户管理系统_2.0.0.zip`

## 注意事项

- 填写软件全称和版本号时务必仔细，填写错误会导致补正
- 合作开发协议的签订日期要在开发完成日期前3个月以上（建议6个月前）
- 截图数量和质量对说明书页数影响较大，建议提前准备充足截图
- 代码不足3000行时全量提交，不足60页时提交全部
- 所有脚本通过 `config.yaml` 配置，无需修改脚本代码
- 所有脚本使用统一日志系统，提供详细错误信息和解决方案

## 常见问题

### Q: 代码不足3000行怎么办？
A: 全量提交所有代码。可补充详细注释、增加空行提高可读性、拆分长行。

### Q: 说明书不足60页怎么办？
A: 可从以下方面扩展：详细描述每个模块、增加截图数量、增加案例分析、扩展常见问题部分。

### Q: 可以删除代码中的敏感信息吗？
A: 可以。删除硬编码的密码、密钥，替换为占位符，删除真实数据库连接字符串。

### Q: 代码注释必须是中文吗？
A: 不强制要求。建议主要注释用中文便于审查，变量名/函数名用英文符合编程规范。

### Q: 一次性通过不下证要多久？
A: 2个月可下证。建议提前3个月以上申请，避免补正影响下证时间。
