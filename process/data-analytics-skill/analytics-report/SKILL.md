# Analytics Report

## 任务目标
- 本 Skill 用于：规范数据分析报告输出标准
- 能力包含：标准报告结构、读者分层策略、输出模板、撰写规范
- 触发条件：需要输出分析报告、呈现分析结论时

## 标准报告结构

### 核心原则：结论前置

**为什么结论前置？**
- 决策者时间有限，需要快速获取关键信息
- 让读者带着结论看细节，更有针对性
- 体现分析师的专业性和价值

## 读者分层与策略

### 四类读者模型

| 读者类型 | 特征 | 关注重点 | 报告策略 |
|----------|------|----------|----------|
| A类 | 高层+了解业务 | 宏观+业务痛点 | 结论前置，趋势图表为主 |
| B类 | 执行层+了解业务 | 微观+运营问题 | 详细分析过程，细节数据 |
| C类 | 高层+不了解业务 | 战略+方向诊断 | 背景详述，结论明确 |
| D类 | 新人+学习目的 | 业务细节+指标定义 | 背景充分，逻辑清晰 |

### 针对A类读者（决策层）
```
标题：XX业务诊断与战略建议
结构：
- 结论与建议（占40%篇幅）
- 业务背景简述（1-2段）
- 核心指标监控数据（趋势图）
- 竞品对比（如有）
- 风险预警（如有）

图表：趋势图、KPI仪表盘、对比图
语言：简明扼要，避免过多技术术语
```

### 针对B类读者（执行层）
```
标题：XX运营问题深度分析
结构：
- 分析目标（明确）
- 数据选择（微观数据）
- 分析过程（详细）
- 结论与建议（可落地）

图表：明细数据、维度拆解图、漏斗图
语言：专业但不晦涩
```

### 针对C类读者（高层外行）
```
标题：XX业务发展分析报告
结构：
- 业务背景详述
- 分析依据充分
- 结论定性总结
- 建议方向明确

图表：直观易懂的可视化
语言：通俗易懂，适当类比
```

### 针对D类读者（学习者）
```
标题：XX业务分析案例
结构：
- 业务背景详述
- 指标定义说明
- 分析步骤详细
- 结论有推导过程

图表：逻辑清晰的流程图
语言：详细解释，便于学习
```

## 撰写规范

### 标题规范
```
格式：[业务/主题]+分析类型+报告
示例：
- "2024年Q1用户留存问题诊断报告"
- "618大促活动效果分析报告"
- "竞品对比分析月报"
```

### 数字使用规范
| 场景 | 规范 | 示例 |
|------|------|------|
| 数量 | 使用阿拉伯数字 | 1000、52.5% |
| 百分比 | 保留1-2位小数 | 52.3% |
| 金额 | 注明单位 | 100万、1.5亿 |
| 范围 | 使用"至"或"-" | 10%-20% |
| 对比 | 明确基准 | 环比+15%、同比-8% |

### 图表规范
```
图表必备元素：
1. 标题：清晰描述图表内容
2. 图例：说明数据系列
3. 坐标轴标签：注明单位和含义
4. 数据来源：注明数据出处
5. 注释：解释异常数据

图表编号：图1、图2、表1、表2
图表引用：如图1所示、如上表所示
```

### 语言风格
| 要素 | 要求 |
|------|------|
| 简洁 | 删除冗余词汇 |
| 准确 | 用词精准，避免模糊 |
| 客观 | 基于数据，避免主观 |
| 可行动 | 结论要可执行 |

### 常见问题

**问题1：结论不明确**
```
❌ 不好：用户增长有所提升
✅ 更好：2024年Q1新用户环比增长15%，主要得益于XX渠道
```

**问题2：建议不可落地**
```
❌ 不好：建议加强运营
✅ 更好：建议在XX渠道增加30%投放预算，预计带来X万新增用户
```

**问题3：数据与结论脱节**
```
❌ 不好：数据显示A产品在涨
✅ 更好：A产品销量环比+25%，主要受益于XX功能上线（贡献度40%）
```

## 注意事项
- 报告是给人看的，要考虑读者背景
- 没有放之四海的标准，只有是否满足受众需求
- 分析师需要清楚：谁看报告、读者期望什么、信任度如何
