## 概览

竞品分析是从零开始规划项目的重要环节，通过系统化研究同类产品，提取设计要点，避免重复造轮子，同时找到差异化方向。

## 竞品选择

### 选择标准

| 标准 | 说明 | 权重 |
|------|------|------|
| **市场地位** | 头部产品/新兴产品/细分市场 | 高 |
| **用户规模** | 用户量级（百万/十万/万） | 中 |
| **技术方案** | 与目标技术栈的相关性 | 高 |
| **开源程度** | 开源/闭源/部分开源 | 中 |

### 选择数量

- **基础分析**：2-3 个竞品
- **完整分析**：4-5 个竞品
- **警惕**：不要超过 5 个，分析量过大容易迷失

### 示例竞品列表

```
效率工具类：
├── uTools          # 头部产品，500万+用户
├── ZTools          # 竞争产品，差异化定位
├── Quicker         # 鼠标操作特色
└── Alfred (Mac)    # macOS 代表作
```

## 分析维度

### 1. 产品定位

| 维度 | 说明 |
|------|------|
| 产品定位 | 一句话描述产品核心价值 |
| 核心理念 | 产品的设计哲学 |
| 目标用户 | 主要用户群体 |
| 市场地位 | 市场份额/用户量级 |

### 2. 核心功能

| 维度 | 说明 |
|------|------|
| 内置功能 | 产品自带的核心功能 |
| 特色功能 | 与竞品区分的功能 |
| AI 功能 | AI 相关能力 |

### 3. 技术架构

| 维度 | 说明 |
|------|------|
| 底层框架 | Electron/Tauri/其他 |
| 前端技术 | Vue/React/Svelte |
| 插件语言 | JS/TS/Rust |
| 跨平台 | Windows/Mac/Linux |

### 4. 插件系统

| 维度 | 说明 |
|------|------|
| 插件定义格式 | JSON/其他 |
| 插件 API | 暴露哪些能力 |
| 插件加载机制 | 热重载/懒加载 |
| 开发体验 | 文档/调试工具 |

### 5. 生态与社区

| 维度 | 说明 |
|------|------|
| 插件数量 | 市场插件数量 |
| 社区活跃度 | 论坛/Discussions 活跃度 |
| 官方支持 | 文档/技术支持 |

### 技术差异化

## 使用示例

### 示例：效率工具竞品分析

**竞品选择**：uTools、ZTools

**分析结果**：

| 维度 | uTools | ZTools | Corelia |
|------|--------|--------|---------|
| 底层框架 | Electron | Delphi+CEF | Tauri |
| 插件数量 | 500+ | 20+ | MVP 8个 |

**可借鉴点**：
- uTools: $schema 支持、toggle 功能
- ZTools: PouchDB、剪贴板历史

**差异化方向**：
- Tauri 替代 Electron（性能优势）
- WASM 补丁层（AI 能力）
