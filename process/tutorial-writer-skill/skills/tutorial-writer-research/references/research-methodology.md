# 资料搜索方法论

> 本文档定义了教程撰写过程中资料搜索的完整方法论，包括搜索策略、质量判断、归档规范和索引管理。

---

## 1. 搜索策略框架

### 1.1 关键词设计原则

好的搜索关键词是找到高质量资料的关键。

| 原则 | 说明 | 示例 |
|------|------|------|
| **具体化** | 用精确术语替代模糊描述 | "RAG chunking strategy" > "RAG 怎么分块" |
| **多角度** | 从不同维度设计关键词组 | 技术/评测/实战/对比 各一组 |
| **时效性** | 加入年份限定获取最新信息 | "2025 2026" 作为必选词 |
| **英文优先** | 技术资料英文质量通常更高 | "vector database benchmark 2025" |

### 1.2 标准搜索模板

每组章节建议执行以下搜索组合：

```
搜索组 A（核心技术）："{技术名词} {核心概念} tutorial guide 2025 2026"
搜索组 B（最佳实践）："{技术名词} best practices production 2025"
搜索组 C（对比评测）："{技术名词} comparison benchmark 2025 2026"
搜索组 D（问题解决）："{技术名词} common problems troubleshooting"
搜索组 E（中文补充）："{技术名词} 教程 实战 2025"（如有优质中文资源）
```

### 1.3 搜索执行规范

| 规范项 | 要求 |
|-------|------|
| 最少搜索次数 | 每章至少 3 次 WebSearch（不同关键词组） |
| 每次结果数量 | 5 条（默认），不足时扩展到 10 条 |
| 结果筛选 | 每次搜索至少深入查看 2-3 个链接内容 |
| 并行搜索 | 不同关键词组的搜索可并行执行以提高效率 |

## 2. 资料质量五级分类

### 2.1 分类标准

| 等级 | 类型 | 典型来源 | 可信度 | 使用优先级 |
|------|------|---------|--------|----------|
| T1 | 官方文档 | 项目官方文档、API Reference、RFC | 最高 | 首选 |
| T2 | 权威教程 | GitHub Star > 1K 的完整教程、知名博客系列 | 高 | 次选 |
| T3 | 评测数据 | Gartner/McKinsey/Stanford 等机构报告、学术论文 | 高 | 数据支撑 |
| T4 | 社区分析 | CSDN/掘金/Medium 等平台的高质量分析文章 | 中 | 补充视角 |
| T5 | 对比汇总 | 多源对比文章、选型指南 | 中 | 快速了解全貌 |

### 2.2 排除标准（不入库）

以下类型的资料不应归档到参考库：

| 排除原因 | 示例 | 处理方式 |
|---------|------|---------|
| 信息严重过时 | 2020 年前的技术文章且无后续更新 | 忽略 |
| 来源不明 | 无作者、无日期、无出处 | 忽略 |
| 内容过于碎片化 | < 300 字的微博/推文摘要 | 忽略 |
| 纯广告/营销 | 产品推广软文无实质技术内容 | 忽略 |
| 纯个人观点无依据 | "我认为 XX 最好"无数据支撑 | 忽略 |
| 重复已有资料 | 与已归档资料高度重叠（>80%） | 忽略 |

### 2.3 质量快速检视清单

对每个候选资料执行 30 秒快速检视：

- [ ] 发布/更新时间是否在 2 年内？
- [ ] 是否有明确的作者或发布机构？
- [ ] 是否包含可验证的数据或代码？
- [ ] 内容长度是否足够有实质信息（>500 字）？
- [ ] 是否与当前章节的主题直接相关？

**通过 ≥ 4 项 → 归档**
**通过 2-3 项 → 可选归档（补充视角时）**
**通过 < 2 项 → 排除**

## 3. 资料归档规范

### 3.1 目录分类映射

| 资料主题内容 | 归档目录 | 对应章节 |
|------------|---------|---------|
| 大模型基础（幻觉、限制、能力） | `01-大模型基础/` | 认知篇 |
| RAG 核心原理（架构、组件、设计） | `02-RAG核心原理/` | 认知篇 + 进阶篇 |
| 技术对比与选型（框架/模型/数据库对比） | `03-技术对比与选型/` | 基础篇 |
| 工具链与环境（安装/配置/部署） | `04-工具链与环境/` | 基础篇 Ch3 |
| 数据处理与索引（分块/Embedding/存储） | `05-数据处理与索引/` | 基础篇 Ch4 + 进阶篇 Ch6 |
| 检索策略与优化（BM25/HyDE/Rerank） | `06-检索策略与优化/` | 基础篇 Ch5 + 进阶篇 Ch6 |
| 生成与提示工程（Prompt/Self-RAG/对话） | `07-生成与提示工程/` | 基础篇 Ch5 + 进阶篇 Ch7 |
| 评估体系（指标/框架/测试集） | `08-评估体系/` | 进阶篇 Ch8 |
| 生产部署与运维（架构/Docker/K8s/监控） | `09-生产部署与运维/` | 生产篇 Ch9-Ch11 |
| 高级范式（Graph RAG/Agentic/多模态） | `10-高级RAG范式/` | 前沿篇 Ch12-Ch13 |
| 行业案例与趋势（落地案例/MCP/Agent） | `11-行业案例与趋势/` | 前沿篇 Ch13 |

### 3.2 文件命名规范

```
{两位数序号}-{中文描述}.md

示例：
├── 01-幻觉率基准与模型排行.md
├── 02-企业RAG采纳数据与趋势.md
└── 03-RAG系统设计完整指南.md
```

序号按主题内顺序递增，不跨主题编号。

```markdown

> 来源: {主页面 URL}

{用 3-5 句话概括资料的核心内容和价值}

{提取可直接引用的关键数据和论点，标注原始出处}

{本资料可用于哪些章节的哪些小节}

```

## 4. 索引管理

### 4.1 资源索引文件结构

`00-资源索引与链接.md` 是整个资料库的主索引，记录所有在线资源的主页面链接。

```markdown
# 资源索引与链接

> 最后更新: {YYYY-MM-DD}
> 总计: {N} 份资料

## T1 梯队：官方文档与权威来源

| 名称 | URL | 归档文件 | 对应章节 |
|------|-----|---------|---------|
| LangChain 官方文档 | https://... | 04-工具链与环境/xx.md | Ch3 |
| ... | ... | ... | ... |

## T2 梯队：高质量教程

| 名称 | URL | 归档文件 | 对应章节 |
|------|-----|---------|---------|
| ... | ... | ... | ... |

（依此类推 T3-T5 梯队）
```

### 4.2 索引更新时机

每次新增资料后，必须同步更新索引文件：
- 在对应梯队中追加一行记录
- 更新总计数量和最后更新时间
- 保持梯队内按字母顺序排列

## 5. 搜索效率技巧

### 5.1 批量搜索模式

对于同一章节的多组关键词，采用并行搜索：

```
并行执行（同时发起）:
  ├── WebSearch("RAG chunking strategy 2025")
  ├── WebSearch("text splitting best practices 2025 2026")
  └── WebSearch("RAG 分块策略 对比 2025")

→ 收集所有结果 → 统一筛选 → 批量归档
```

### 5.2 搜索结果去重

不同关键词可能返回相同或高度重叠的结果。去重策略：

- **URL 去重**：相同 URL 只保留一次
- **域名去重**：同一域名的文章保留最全面的版本
- **内容去重**：标题相似度 > 80% 时只保留一篇

### 5.3 深度抓取策略

对于高质量来源（T1/T2），使用 WebFetch 抓取完整页面内容而不仅仅是搜索摘要：

```
WebSearch → 筛选出 T1/T2 结果 → WebFetch 获取完整内容 → 提取关键部分存档
```

这样可以确保归档的资料包含足够的细节供后续撰写引用。
